# OpenClaw 标准化规范

## 01 OpenClaw 文件路径标准
```bash
请读取以下文档，并将其作为当前 OpenClaw 容器环境的标准路径规范来落地执行：
https://github.com/aoliyougei/OpenClaw-Parameters/blob/main/01.OpenClaw%20%E6%96%87%E4%BB%B6%E8%B7%AF%E5%BE%84%E6%A0%87%E5%87%86
具体要求：
1. 提炼文档中的核心路径标准，包括：标准根目录、配置文件路径、状态目录、workspace、插件目录、npm 依赖目录、bin 目录、临时文件目录、持久文件目录、环境变量规则
2. 检查当前环境与该标准的差异
3. 给出“当前状态 vs 标准状态”的对比清单
4. 在不破坏现有运行的前提下，尽可能将环境整理为该文档要求的标准路径结构
5. 如需创建目录、迁移路径、补充环境变量或调整配置，请直接执行可安全执行的部分；涉及风险或重启的部分，先明确列出
6. 将适合作为长期规则的内容提炼写入  ⁠AGENTS.md⁠ ，确保后续默认遵循
7. 完成后输出：
 ⦁已采用的路径标准
 ⦁已修改的文件和目录
 ⦁仍需人工执行的步骤
 ⦁验证方法
```

## 02 启用向量模型、缓存和混合搜索
```bash
请读取以下文档，并将其作为当前 OpenClaw 的记忆检索配置标准来落地执行：
https://github.com/aoliyougei/OpenClaw-Parameters/blob/main/02.OpenClaw%20%E5%90%AF%E7%94%A8%E5%90%91%E9%87%8F%E8%AE%B0%E5%BF%86%E3%80%81%E7%BC%93%E5%AD%98%E4%B8%8E%E6%B7%B7%E5%90%88%E6%90%9C%E7%B4%A2%E6%93%8D%E4%BD%9C%E6%8C%87%E5%8D%97
具体要求：
1. 提炼文档中的核心配置要求，包括向量记忆、缓存/文件监听、混合搜索的启用方式
2. 检查当前环境是否已经具备这些配置
3. 输出“当前状态 vs 文档标准”的差异清单
4. 在不破坏现有运行的前提下，尽可能将当前 OpenClaw 配置调整到文档描述的目标状态
5. 如需修改配置文件，请直接修改可安全修改的部分；如涉及 API Key、重启、模型选择或高风险改动，请明确说明
6. 配置完成后，执行完整验证，包括：
 ⦁向量语义召回是否生效
 ⦁缓存/文件监听是否生效
 ⦁混合搜索是否生效
7. 将适合作为长期规则的部分提炼写入  ⁠AGENTS.md⁠ ，确保后续默认遵循
8. 最终输出：
 ⦁已采用的配置
 ⦁已修改的文件
 ⦁已完成的测试项
 ⦁尚需人工处理的步骤
 ⦁验证方法
目标不是总结文档，而是让当前 OpenClaw 尽可能达到文档中描述的实际效果。
```


## 03 记忆最佳实践
```bash
请阅读以下文档，并将其作为你的记忆工作规范：
https://github.com/aoliyougei/OpenClaw-Parameters/blob/main/03.OpenClaw%20%E8%AE%B0%E5%BF%86%E6%9C%80%E4%BD%B3%E5%AE%9E%E8%B7%B5
具体要求：
1. 提炼其中适合作为长期执行规则的内容，写入  ⁠AGENTS.md⁠ 
2. 按文档结构初始化或整理  ⁠MEMORY.md⁠ 
3. 按文档结构初始化当天的  ⁠memory/YYYY-MM-DD.md⁠ 
4. 结合你当前的角色定位，补充  ⁠Role Alignment⁠  和  ⁠Role Notes⁠  的使用规则
5. 后续在记忆写入、蒸馏和周期维护时默认遵循该文档
6. 如果文档中有与当前角色不匹配的部分，请按当前角色做适配，而不是机械照抄
完成后告诉我你采用了哪些规则、初始化了哪些文件，以及后续会如何执行。
```
